# 开干入口：新会话 / 新任务 + 流内选项目

**日期：** 2026-07-22

**状态：** 规格已确认，待写实现计划

**范围：** Code 侧栏「开始」动作、Draft Session / 新任务流内的项目上下文选择、仓库发现；Teambition 认领入口本期关闭

**关系：** 修订并收窄 `2026-07-22-workspace-project-session-ia-design.md` §6.1 / §8.3 中「打开文件夹与新会话并列一等入口」的表述。Workspace / Project / Session 对象边界、双投影、Project 详情点击规律仍以该文为准。

**参考：** Cursor New Agent + 会话内 Search folders/repos；Codex Desktop thread 落点；Hermes Desktop 有界 repo 发现；craft-agents name-first 建壳。

## 1. 决策摘要

- **双主开始动作**：侧栏顶栏为 `[＋ 新会话] [＋ 新任务]`，其后才是搜索 / 收起。去掉 Folder 并列主按钮与「更多创建」折中方案。
- **选项目 / 新建项目只发生在「新会话」与「新任务」流程内部**，不作为侧栏独立常驻入口（不钉在双投影旁，也不再作为顶栏第三 CTA）。
- **先开容器，再绑项目**：对齐 Cursor / Codex。
- **建项终点**：绑定该 Project 的 Draft Session（新会话）或任务创建流内的会话（新任务）；不再默认跳 Project 详情。详情仍由点击项目名进入。
- **仓库发现**挂在流内项目选择器；默认不扫 `$HOME`。
- **Teambition 认领本期关闭**（UI 不可达；候选短文案后续：`领取 TW`）。

## 2. 背景与问题

当前侧栏同时存在：无字 Folder、双投影旁「新建项目」、Board「新增任务 / Teambition」——多套「开始」且项目绑定时机不一致。正确心智是：**先选要开的是会话还是任务，再在该流程里选/建项目。**

## 3. 目标与非目标

### 3.1 目标

1. 顶栏固定双 CTA：`新会话`、`新任务`。
2. 选择已有项目、新建项目（含路径浏览 / 发现）**仅**出现在上述两流内部的共享选择器。
3. 新会话可暂不绑项目；新任务必须选或建项目。
4. tab 行只切换投影；TW 认领对本期不可见。

### 3.2 非目标

- 本期上架 Teambition 认领
- Hermes worktree / branch lane、启动全盘扫描、自动批量建壳
- 修改「Task 必须属于 Project」业务规则
- README / AGENTS.md（须另获许可）

## 4. 信息架构

### 4.1 侧栏顶栏

```text
[＋ 新会话] [＋ 新任务]     [搜索] [收起]
```

- **新会话**：创建 Draft Session → 主区展示项目上下文选择器（可跳过为「无项目」）。
- **新任务**：进入任务创建流 → **必须先**经同一套项目选择器（选已有 / 新建 / 浏览路径 / 发现）；选定 Project 后再填任务字段并提交。
- 去掉与二者并列的 Folder 主按钮。
- Board 顶栏「新增任务」可收敛为次要入口或移除双份（实现计划二选一；行为须与顶栏「新任务」同一套流）。
- `⌘O`：在当前新会话 Draft（或打开选择器的上下文）中聚焦「浏览…」/ 打开系统选目录；不单独充当第三主按钮。

### 4.2 双投影行

```text
[最近会话｜项目]
```

- **只切换投影**，右侧不放「新建项目」。
- 「项目」视图 `⋯` **仅**管理类：显示已归档、添加扫描目录 / 刷新发现。**不含**「新建项目」（新建只走 §4.1 两流）。

### 4.3 共享：项目上下文选择器

新会话 Draft（未绑定或用户打开换绑）与新任务流第一步共用同一组件，分区：

| 分区 | 新会话 | 新任务 |
|------|--------|--------|
| Recents | ✓ | ✓ |
| 已有项目 | ✓ | ✓（必选其一路径） |
| 发现 | ✓ | ✓ |
| 浏览…（系统选目录 → openOrCreateByPath） | ✓ | ✓ |
| 新建项目…（name-first） | ✓ | ✓ |
| 无项目 | ✓ 允许 | ✗ 不允许 |

- **新建项目…**：名称必填；工作目录可选。提交后创建 Project 并绑定**当前流**（会话 Draft 或待建任务），不跳详情。
- **浏览… / 发现项**：`openOrCreateByPath` 后绑定当前流。
- 新会话绑定后采用 Project 有效 cwd；Draft 未发送即丢弃语义不变。

### 4.4 空「项目」视图

不提供独立「新建项目」主按钮。引导文案指向顶栏：

> 用「新会话」或「新任务」开始，并在流程中选择或创建项目。

可选次要链接：添加扫描目录…（只影响发现，不建壳）。

### 4.5 Teambition 认领 — 本期关闭

隐藏一切认领入口；后端可保留。后续再定上架位置与短文案（`领取 TW`）。

## 5. 仓库发现

| 项 | 规则 |
|----|------|
| 默认 | 不扫 `$HOME`；无扫描根则发现区空并引导添加 |
| 深度 | 默认 3（1–5 可配） |
| 识别 | 含 `.git` 为 repo root |
| 时机 | 打开选择器时懒扫 + 缓存 |
| 点击 | 同 openOrCreateByPath，绑定**当前**新会话或新任务流 |

## 6. 成功标准

1. 顶栏可见且仅此两个开始 CTA：新会话、新任务。
2. 用户无法在侧栏「单独新建项目」而不经过会话/任务流。
3. 新任务在未选定 Project 前不能提交。
4. 无字 Folder 不再是开干主入口；TW 认领不可见。
