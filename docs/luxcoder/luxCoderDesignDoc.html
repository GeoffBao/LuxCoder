产品定位

LuxCoder 不是 ChatGPT UI，也不是单纯的 Claude Code GUI

🎯
目标用户
软件部全体研发工程师、架构师、测试工程师、项目负责人、AI 平台团队
🏭
部署形态
Electron 桌面应用 + 企业 Agent Server，内部分发，支持 macOS / Windows
🔗
核心集成
Teambition 双向同步、Git worktree、RAG 知识库、MCP 工具、spec-kit 流程
🧠
Chat 内核
Claude Agent SDK（Phase 1）+ Hermes provider（Phase 2）— 具备 Skills、MCP 工具调用、RAG 知识检索能力的研发助理
⚙️
Code 内核
Claude Agent SDK — 文件读写、Shell 执行、工具调用、流式事件
📋
Cowork 内核
Claude Agent SDK + LuxCoder TaskOrchestrator — 五阶段看板 + Teambition 实时双向同步

03

Chat 模式

Phase 1：复用现有 Claude provider · Phase 2：接入 Hermes（企业内网 HTTP API）

内核：Claude Agent SDK
Phase 1 复用现有 Claude provider（和 Code 模式共用），具备 Skills、MCP 工具调用、RAG 知识检索能力。Phase 2 接入 Hermes provider（企业内网 HTTP API），不改 agent loop。
知识包 Chips
Skills · camera_hal MCP · perfetto RAG · sm8845_isp
用户手动选择 + Hermes 自动记忆，支持项目级 ACL 权限控制。
转 Code / 转 Cowork
Chat 结论可一键转成 Code run（带上下文），或转成 Cowork 任务卡（自动填充描述和知识包）。
来源引用
回答中展示知识来源（文件路径、文档标题、RAG chunk），支持点击跳转。禁止越权检索其他项目知识包。

Cowork 设计要点
🔄 研发不再需要打开 Teambition
任务从 TB 自动同步进来，工程师在 LuxCoder 内完成全部操作：查看任务 → 生成 Spec → AI 编码中 → 验收归档。状态、Spec URL、经验文档自动回写 TB，TB 只作为数据存档后台。
⑂ 独立 Git Worktree
每张 编码中 卡片对应一个独立 git worktree，agent 在隔离环境里运行，不影响主分支。多任务并行时互不干扰，完成后 PR 合并。
📝 五阶段 Spec-kit 流程
任务 → 需求 Spec（AI 生成 + 人工确认）→ 方案 Spec（AI 生成 + 人工确认）→ AI 编码中（agent 执行）→ 验收归档（结果回写 TB）。每阶段产物存档到 TB 自定义字段。
📋 任务类型分流
需求 → 完整六阶段（方案设计 + TB 审核）；Bug/简单任务 → 直接 编码中（跳过方案设计 + TB 审核），提交 Gerrit 时附带最小化 fix description。
📝 方案文档模板
模板植入 LuxCoder：问题描述 → 根因分析 → 方案对比 → 实施计划 → 测试计划。AI 辅助填充草稿，人工确认后上传 TB 附件存档。
👤 个人看板视图
只显示责任人是自己的任务。TB 上 Leader 分配任务时指定责任人，LuxCoder 自动拉取 assignee = 我的任务。
