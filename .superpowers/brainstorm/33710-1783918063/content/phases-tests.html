实施阶段、测试矩阵与完成标准

每一阶段都可独立验证；最终删除旧 Work 后，Project/Kanban 成为唯一入口。

纵向实施阶段
P0

边界清理

锁定 source commit、版本契约、依赖、删除清单；保留用户未提交改动。

P1

Shared

Project/Task/Protocol 完整对齐；storage 与 renderer-safe exports 分离。

P2

Runtime

Session 字段、Project/Task IPC、Preload、TaskRunner/Host、事件恢复。

P3

Kanban UI

Atoms、Board/Column/Tile、Project 面板、拖拽、编辑器、子任务与运行态。

P4

Teambition

Picker、binding、徽标、claim、sync progress、status update、冲突状态。

P5

收口

删除旧 Work/LangGraph/THO；去除 @ts-nocheck/TODO；文档与版本同步。

最终验收清单

用户可见能力

  • Workspace 内创建、编辑、归档、删除 Project
  • 浏览 Project Sessions、Assets、Memory、Settings
  • 普通 Session、quick-add task、spec task 均可进入看板
  • 拖拽、筛选、列表切换、TaskEditor、子任务管理可用
  • 运行/暂停/恢复/停止/修复/重启恢复可用
  • Teambition 领取、绑定、同步和冲突提示可用

工程质量门槛

  • bun run typecheck 通过
  • shared/main/renderer/Teambition 测试通过
  • bun run dev 冷启动无白屏、无 Node fs renderer 错误
  • 生产 renderer/main/preload build 通过
  • 没有 @ts-nocheck、迁移 TODO、旧 Work 调用
  • 只修改迁移相关文件，版本与文档按项目规则同步
✓
