设计 §3 · 边界 / 测试 / 后续

本轮做什么、不做什么，以及验收点。

本轮做（P0）

  • 左栏序：自动任务 → Agent 技能 → Agent 专家 → 项目中心（模块区末）→ 工作区会话
  • 项目中心 / Agent 专家：入口行 → activeView 全屏 Hub（对齐技能）
  • 侧栏不再展开项目列表；Hub 点卡片进看板；详情挂默认专家
  • 专家目录壳 + 文件系统包（IDENTITY/SOUL/RULES + expert.json 引用）
  • 修订/取代今早 IA 里「项目可折叠展开」条款

本轮不做

  • 专家 runtime 注入（orchestrator / Skill 自动装载）
  • Kimi 式蜂群编排 / 主 Agent 调度子 Agent UI
  • 飞书/Discord bot 绑定配置 UI（仅 channelBindings:[] 占位）
  • 每专家 USER.md；探索/规划专家角色（可列 P1）
  • 统一 Module Registry 大重构

测试（BDD 向）

  • 点「项目中心」→ 主区 Hub，侧栏无展开列表
  • Hub 选项目 → 看板过滤；详情可选默认专家并持久化
  • 点「Agent 专家」→ 专家网格；内置 slug 可见
  • Chat 模式不显示项目中心 / Agent 专家
  • 纯函数：activeView 路由、defaultExpertId 读写、expert.json 解析

后续钩子（写入 spec，不实现）

  • 蜂群：主 Agent + 专家子 Agent，上下文隔离（对齐 Kimi Swarm / Code sub-agents）
  • Bot：Expert ↔ Feishu accountId / Discord app；@路由；群内 Agent@Agent
