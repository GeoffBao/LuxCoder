# LuxCoder v0.1.96 更新

相对 `v0.1.94`：开干入口与项目选择体验、OpenRouter、默认 Pi 内核，以及一批 Code/设置 UI 修复。

## 新功能

- **开干入口 — 新会话 / 新任务** — Code 侧栏顶栏改为双 CTA；流内选项目（浏览 / 发现 / 新建）；新任务必须先选项目；支持 ⌘O 浏览文件夹。(#14)
- **OpenRouter 供应商** — 新增 OpenAI 兼容的 OpenRouter 渠道类型，可接入其聚合模型。
- **默认仅 Pi 内核** — 默认关闭 Claude 内核，仅保留 Pi；需要 Claude 时通过 `CLAUDE_RUNTIME_ENABLED` 开关启用。
- **用户消息右对齐** — Chat / Code 用户气泡右对齐，并向导 / 教程内容全面更新。(#13)
- **内置头像** — 用户档案改用打包头像资源，替换 emoji 选择器。

## Bug 修复与 UI

- **新会话 / 新任务不再误弹系统选文件夹** — 修正 ProjectContextPicker 对历史 browse token 的回放；Draft 未绑项目时默认展开选择器。(#18)
- **项目选择器更紧凑** — 列表只显示项目名（完整路径进 tooltip），动作钉底；Draft 触发器改成紧凑 chip。(#15)
- **思考模式开态对比更清晰** — 拆清「生成 vs 展示」两套状态，避免开关态混淆。(#16)
- **Agent 技能文案** — 「PROMA 内置」改为「系统内置」。(#17)
- **设置页精简** — 移除模型配置页官方供应推广卡片；默认主题跟随系统。(#12)

## 其他

- **发布 tag 与版本校验** — CI 在构建前强制校验 `vX.Y.Z` tag 与 `apps/electron/package.json` 版本一致。

## macOS 打开说明（重要）

当前 Release **未配置 Apple Developer ID 签名/公证**。从浏览器下载后，macOS Gatekeeper 可能误报：

> “LuxCoder” is damaged and can’t be opened.

**这不是安装包损坏。** 安装到 Applications 后执行：

```bash
xattr -cr /Applications/LuxCoder.app
open /Applications/LuxCoder.app
```

若还在 Downloads 里的 `.app` / `.dmg`，先对对应路径跑同样的 `xattr -cr`。
