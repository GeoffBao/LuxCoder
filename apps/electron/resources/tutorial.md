# LuxAgents 使用指南

版本：v1.0 · 2026 年 6 月

---

## 产品定位

LuxAgents 是面向企业研发组织的 AI Agent 桌面工作台，将 Chat、Code、Work 三种工作模式整合在一个统一桌面应用中。

**核心定位**：不是 ChatGPT UI，也不是单纯的 Claude Code GUI，而是为研发工程师、架构师、产品经理和测试工程师设计的 AI 协作平台。

| 模式 | 定位 | 状态 |
|------|------|------|
| **Chat** | AI 研发助理 · 多模型对话 · 知识检索 | 可用 |
| **Code** | AI 编程工作台 · Claude Agent SDK 驱动 · 文件读写 | 可用 |
| **Work** | 协作看板 · Teambition 双向同步 · 六阶段研发流 | 即将上线 |

---

## 快速开始

### 第一步：配置 AI 供应商

进入 **设置 → 模型配置 → 添加配置**，支持以下渠道类型：

- **Anthropic**（Claude 系列）— 支持 Chat 和 Code 两种模式，推荐作为主力渠道
- **DeepSeek / Kimi**（Anthropic 协议兼容）— 同样支持 Code 模式
- **OpenAI / 通义 / 智谱**（OpenAI 协议）— 仅支持 Chat 模式
- **自定义端点**（OpenAI 兼容格式）— 接入企业内网模型服务

填写 API Key 后点击右上角创建即可完成配置。配置后在 **设置 → Agent 供应商** 中开启需要作为 Code 模式使用的渠道。

### 第二步：Windows 环境准备（仅 Windows）

Code 模式在 Windows 上需要 Git Bash 或 WSL 才能执行 Shell 命令。首次启动时会引导完成检测和安装，跟随向导操作即可。

### 第三步：创建工作区（Code 模式）

Code 模式基于**工作区**组织工作。工作区对应一个本地项目目录，每个工作区独立管理：

- **MCP 服务配置**：工具扩展（文件系统、数据库、API 等）
- **Skills 列表**：可复用的工作流提示词
- **会话历史**：该工作区下的所有对话

在 **设置 → Agent 工作区** 中创建工作区，关联到你的项目目录即可开始使用。

## Chat 模式

Chat 模式是多模型对话助理，适合需求分析、技术咨询、文档撰写等非执行类工作。

### 主要能力

- **多供应商并行**：可配置多个渠道，按需在对话中切换模型
- **多模态输入**：支持上传图片、PDF、Office 文档、代码文件等附件
- **并排对比**：同一问题向多个模型发送，对比回答质量
- **思考模式**：启用 Claude 扩展思考，处理复杂推理任务
- **上下文控制**：手动插入上下文分割线，精确管理有效上下文范围

### 上下文管理建议

每个对话聚焦一个主题，不建议在同一对话中混入无关问题。上下文越简洁，回答速度越快、准确度越高、Token 消耗越少。

## Code 模式

Code 模式由 Claude Agent SDK 驱动，具备完整的文件读写、Shell 执行、工具调用能力，是 AI 编程协作的核心入口。

### 工作区文件结构

```
工作区目录/
├── .context/          # Agent 协作上下文（设计文档、进度追踪、踩坑记录）
├── workspace-files/   # 工作区持久文件
└── [项目文件...]
```

每个**会话**对应一个独立的子目录，用于存放该次任务的临时文件。

### 文件组织原则

| 文件类型 | 存放位置 | 说明 |
|----------|----------|------|
| 临时任务文件 | 会话文件夹 | 该次任务专用，完成后可清理 |
| 跨会话复用文件 | 工作区文件夹 | 项目文档、设计规范、数据参考 |
| 附加外部目录 | 附件方式 | 关联现有项目目录，不占用额外磁盘空间 |

**实践要点**：工作区可以附加外部目录（如 Git 仓库、Obsidian vault、数据文件夹），Agent 可以读写这些目录中的文件，实现与现有工具链的无缝协作。

### 上下文提示

表达任务时，给 Agent 完整的上下文比简短的指令效果好得多：

> ✅ 好的表述：「当前工作区有 @销售报表.xlsx，其中 sheet1 是本月数据，成本项在 @成本表.xlsx 的 sheet2，字段对应关系见 @说明.md。请先读取文档，帮我计算各产品线毛利率，结果写入销售报表 sheet1。有不确定的地方先停下来确认，完成后给我一份分析方法摘要。」
>
> ❌ 不够好：「帮我做下报表」

### 权限控制

Code 模式提供三级权限：

| 模式 | 行为 |
|------|------|
| **安全模式** | 仅允许读操作，写操作需逐一确认 |
| **询问模式** | 每次工具调用前弹出确认 |
| **允许全部** | 自动批准所有工具操作（适合信任的自动化任务） |

## Skills 系统

Skills 是可复用的工作流提示词，将团队的最佳实践固化为 Agent 可以自动执行的标准流程。

### 内置 Skills

| Skill | 用途 |
|-------|------|
| `luxagents-coach` | 引导合理使用 LuxAgents，分析你的使用模式并给出优化建议 |
| `skills-creater` | 根据你描述的工作流自动创建新 Skills |
| `find-skills` | 在互联网上搜索适合你场景的 Skills |

在 Code 模式对话框中输入 `/` 可以查看和调用当前工作区的 Skills 列表。

### 创建自定义 Skills

**方式一**：直接对话创建。在 Code 模式下告诉 Agent 你的工作流，让它帮你转换成 Skills：

> 「我需要处理 PR：拉取最新代码 → 检查代码质量 → 生成 Code Review 报告。帮我把这个流程创建成一个 Skills。」

**方式二**：任务完成后沉淀。完成一次效果好的任务后，让 Agent 将刚才的操作流程抽象成 Skills：

> 「刚才的操作流程很好，帮我把它整理成一个可复用的 Skill。」

Skills 创建后会出现在工作区的 Skills 列表中，后续在该工作区的所有会话中均可使用。

### 团队 Skills 分发

- **导出**：在设置中导出配置包（`.luxagents-share` 文件），包含 Skills 和 MCP 配置
- **导入**：收到配置包后双击或在设置中导入即可生效
- **跨工作区复制**：可以在 Code 模式中让 Agent 将 Skills 复制到其他工作区

## MCP 服务

MCP（Model Context Protocol）是 Agent 的工具扩展机制，让 Agent 能访问外部系统和服务。

常见 MCP 用途：

| MCP 类型 | 用途示例 |
|----------|----------|
| 文件系统 | 访问特定目录范围内的文件 |
| 数据库 | 查询公司内部数据库 |
| API 网关 | 调用企业内部服务接口 |
| Teambition | 读写任务、评论（Work 模式核心） |

在工作区设置中配置 MCP 服务，或者直接在 Code 模式中让 Agent 帮你配置：

> 「帮我配置一个访问 ~/projects/backend 目录的文件系统 MCP。」

## 自动化任务

自动化任务允许按计划定时执行 Agent 工作流，适合重复性运营任务。

**创建建议**：最好的时机是在一次手动任务执行成功后，趁热让 Agent 将其转换成定时任务：

> 「刚才的流程执行得不错，帮我把它设置成每天 9:00 自动执行的定时任务。」

自动化任务执行后可以通过飞书推送结果，支持在手机端查看。

---

## 飞书集成

飞书集成让你可以在不打开桌面应用的情况下远程触发和查看 Agent 任务，实现约 80% 的移动端体验。

**支持能力**：

- 在飞书消息中触发 Agent 任务，接收流式卡片响应
- 通过飞书 CLI 打通飞书文档、日程、会议等全套能力
- 自动化任务完成后推送飞书消息通知

在 **设置 → 飞书集成** 中扫码授权即可完成配置。

## 配置管理与团队分发

### 导出配置

在 **设置 → 数据迁移** 中可以导出两种格式：

| 格式 | 说明 | 适用场景 |
|------|------|----------|
| `.luxagents-backup` | 全量备份，包含解密后的 API Key | 个人设备迁移 |
| `.luxagents-share` | 共享包，凭据自动剥离 | 向团队成员分发配置 |

### 导入配置

双击 `.luxagents-backup` 或 `.luxagents-share` 文件，或在设置中手动导入均可。

**推荐做法**：由团队负责人配置好标准工作区（包含 Skills、MCP、渠道）后导出 `.luxagents-share`，分发给所有团队成员，确保团队基础配置一致。

## 最佳实践

### 会话管理

- 每个会话聚焦一个独立任务，完成后开新会话
- 对于跨会话的大型任务，让 Agent 在 `.context/` 目录维护设计文档和进度追踪，新会话时引用该文件继续工作
- 使用 `&` 符号可以在对话中引用其他会话的内容

### Skills 治理

- 只安装经过验证的、解决真实问题的 Skills
- 定期审查工作区 Skills 列表，禁用不再使用的 Skills
- 团队共享 Skills 前先在本地充分测试稳定性

### 模型选择

| 任务类型 | 推荐模型 |
|----------|----------|
| 复杂编程、架构设计 | Claude Opus / Sonnet（Anthropic） |
| 日常问答、文档撰写 | DeepSeek / Kimi（成本更低） |
| 高并发自动化任务 | 按成本预算选择，优先稳定性 |
